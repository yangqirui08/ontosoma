# OntoSoma · 已锁定的关键决策

> 本文记录已拍板的决策 + **为什么**。本体即文档——决策也住在仓里,不漂移。
> 状态:`locked` = 已锁定不再议;`open` = 待定。

---

## D1 · 名字 = OntoSoma `locked`

**决策**:工具名 **OntoSoma**(Onto = 本体/存在 ὄν;Soma = 身体/活体 σῶμα),中文统称「本体」。标语:*你系统的活体本体,永远等于代码*。

**为什么**:
- 退掉旧名 **CaaO(Code as Ontology)**:念不顺、"ontology"太学术、且只命中"代码↔本体"一根支柱。
- 退掉过短的 **Ontos**:体现了"本体"但没体现本体的**特性**。
- `Soma`(活体)写入名字,正是 OntoSoma 区别于一切 spec 工具的气质——**活体 vs 标本**(支柱四)。
- 两个词根都是希腊语,合成像 chromosome/biome,**天然合法的造词**,有分量。
- "准/零漂移"不必塞进名字:**合格的本体天然含零漂移与真源**,该特性由名字内涵 + 标语共同承载。

---

## D2 · 技术栈 = TypeScript `locked`

**决策**:
- 引擎 + web = **TypeScript(Node)**,编译 `tsc`,零/极少运行时依赖。
- 规则/方法论 = **Markdown + AGENTS.md**。
- 解析 = **tree-sitter(WASM)**。

**为什么**:
- 两个标杆 **superpowers 与 openspec 都是 Node/TS**;要够到"那种高度的通用插件",选别的语言是主动偏离已验证栈,还要为分发多付环境税。
- AI 编程工具宿主天然有 Node,npm 是唯一通用分发渠道,**零依赖 = 装上即用 + 无供应链风险**(superpowers 把 714 个 node_modules 砍到 0 即此原则)。
- 用户**全程 Opus 4.8、从不看代码**,语言对用户零成本——选 TS 唯一的"手感代价"也不存在,变成纯赢。
- 系统心脏是"本体 schema",做成类型让派生器与闸全程吃类型安全(选 TS 不裸 JS 的原因)。

---

## D3 · 本体存储 = 文件 lockfile + 嵌入式查询,不上图数据库 `locked`

**决策**:派生本体写成文件、当 lockfile 提交进仓;查询用 **DuckDB/SQLite**(递归 CTE)。**不引入 Neo4j 等需起服务的图数据库**(仅作可选后端留给组织级)。

**为什么**:
- 本体是**从代码派生的缓存/索引,不是系统记录**——不需要事务/ACID/常驻服务。
- 规模 10³–10⁵ 节点,内存遍历微秒级。
- "本体即 lockfile"一招串起时间轴(git 历史)、设计↔实际可 diff、零漂移闸(CI 比对)、PR 可评审。
- 要求用户装图数据库 = 直接干掉"装上即用";superpowers 的零依赖哲学是反复印证的反例。

## D4 · 独立仓,A-CDM 降格为狗粮目标 `locked`

**决策**:在 `D:\Dev\ontosoma` 新建独立 monorepo;**不**在 A-CDM 项目里开发。A-CDM = 第一个狗粮目标 + 可挖的参考原型。

**为什么**:
- **分发**:有自己版本/release/repo 的插件,没法当某 app 的子目录。
- **狗粮要干净**:把 A-CDM 当成它分析的外部目标,才能逼出正确边界(否则 A-CDM 特例漏进工具,通用不了)——编译器要跟被编译程序分开。
- **复用**:以后要扫 A-CDM/GFTM/未来项目,共享工具不能住在某个目标体内。
- **自举即最强 demo**:第一份本体就是 OntoSoma 描述 OntoSoma 自己,只有独立仓做得到。
- 两个标杆(obra/superpowers、Fission-AI/OpenSpec)都是各自独立仓,不寄生使用者项目。
- A-CDM 截图里 `规则闸 / 组成透视 / 语义·对照·源码·图谱` 是真跑通的好东西,新仓里**重做成通用版**,不从零拍脑袋,也不原地改 A-CDM。

## D5 · 跨工具可移植性机制 `locked`

**决策**:AGENTS.md 为基座 + 各 harness 薄清单 + 会话启动自动注入;内容层"描述动作不点名工具",工具名隔离到 per-harness 映射。绝不编辑用户全局配置。

**为什么**:AGENTS.md 已是事实标准(Agentic AI Foundation,2025 末);"会话启动自动注入"是 superpowers 验证过的、整套集成的非协商核心机制;"动作而非工具名"是其跨 6+ harness 不改正文的根本原因。

---

## D6 · 构建路线 = 先传统,搞清楚就立即吃狗粮 `locked`

**决策**:阶段 1 用最常规方式把骨架跑顺(派生器 + lockfile + 闸),**不急着炫**;概念一清,阶段 2 **立即**让 OntoSoma 描述自己并对准 A-CDM 验闭环;阶段 3 再补 web/运行态/分发做成通用插件。

**为什么**:核心闭环(支柱一)与三态对账(支柱四)只有在真代码上才证得了;先传统降低初期不确定性,早狗粮早暴露问题(问题暴露 > 隐藏)。
