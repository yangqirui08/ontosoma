# OntoSoma · 命名、拼写与定位约定

> 名字越做越稳的前提:**自己各处写法一致**,以及**主动澄清定位**,别被同形术语稀释。

## 官方拼写(唯一正确写法)

**OntoSoma** —— 驼峰,O 和 S 大写。

> Official spelling: **OntoSoma**(不是 Onto_Soma / OntoSOMA / Ontosoma Toolkit™ / onto-soma 等变体)。

| 场景 | 写法 |
|---|---|
| 品牌 / 标题 / 正文 | `OntoSoma` |
| npm 包 / 目录 / 包名 | `ontosoma`(全小写) |
| npm 子包命名空间(将来) | `@ontosoma/engine`、`@ontosoma/web` 等 |
| CLI 命令 | `osoma`(好敲)—— 二选一后定死,不混用 |
| 中文 | 统称「本体」 |
| 词源(首次出现时) | OntoSoma(onto- + σῶμα) |

## 定位:这是什么、**不是**什么

**是**:面向 **AI 开发 / 运维 / 迭代** 全生命周期的**代码/系统本体**工具——从代码派生唯一真源模型,保持设计/代码/运行三态零漂移,镜头化、可回放。

**不是**:语义网 / OWL / RDF / 知识图谱建模工具。

> ⚠️ **同形术语风险(必须主动管理)**:"OntoSoma"字面像语义网术语,外人(包括 AI 助手)极易默认我们是 OWL/RDF 工具。**每个对外入口的第一屏都要一句话澄清定位**,否则会被语义网赛道稀释、招来错的人群。

## 展示层语言与文化约定（本体展示层）

> 借鉴 A-CDM CaaO 的成熟实践;两套本体系统共用一套中文词汇,降低心智负担。

**第一原则(根)**:每一处可视展示,必须**同时**满足「对人理解友好」+「对AI准确」——两条都是硬约束,不是取舍。下面的"中文优先 / 英文必须处 / 传统文化"全部由它推导而来(详见 [vision.md §3.5](vision.md))。

**推论一:本体展示层中文优先。** 观看者是中国人 → 中文 + 承载概念的传统文化对人最友好,而 AI 同样能准确理解 → 两轴都满足,是净赢。给人和 AI 做需求分析 / 理解 / 设计的一切展示,默认中文母语化。英文**只**用在"翻译反而损害准确"处:
- 机读 key / 标识符(对象 key、字段名)
- 源代码里的真实标识符(类 / 函数 / 文件名等——它们就是代码真源,改不得)
- 已成行业标准的技术词(npm、AGENTS.md、tree-sitter、lockfile…)

**推论二:派生自代码的对象,用「中文名 + 代码锚」并存。** 派生对象的本名是源码标识符(`deriveOntology`)——只给英文则人看不懂,只给中文则断了代码锚。解法照 A-CDM(中文名 + 机读 key):**标注层**(`ontosoma.annotations.json`,与 lockfile 分离)给每个对象一个中文名 + 一句话;展示层显**中文名**(懂),代码标识符作为**锚**留在悬停 / 详情里(准)。lockfile 保持纯代码派生,标注层是叠加的中文壳。

**恰当融入中国传统文化(适度;承载概念才用,不做装饰):**
- **续存 / 过程 = 物 / 事 = 阴(静)/ 阳(动)** —— 实体 vs 事件的硬范畴(引入实体/过程节点时采用)。
- **判 / 改 / 人 = 天 / 地 / 人 三才** —— 观测 / 改造内核 / 人事 的三分。
- **照镜** —— 代码是唯一真源,展示层只读映射(以镜喻真)。
- **声明 ↔ 代码一致 = 天人感应** —— 不一致即被规则闸拦下。

**与 A-CDM 术语对齐:** 本体 / 对象 / 属性 / 联系(关系)/ 范畴(kind)/ 标签 / 派生 / 对账 / 规则闸 / 零漂移 / 照镜;详情镜头沿用 **语义 · 对照 · 源码 · 图谱**;关系强度沿用 **外键 · 有代码锚 · 软引用**(引入声明态关系时采用)。
